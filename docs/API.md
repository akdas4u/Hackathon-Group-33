# API Reference

Base URL: `/api/v1` (versioned via `Asp.Versioning.Http`, URL segment, default `v1`). All examples below use the scripted demo dataset (release `REL-2002`, "Payments Gateway", branch `release/2.14`).

Every response carries a `correlationId`, generated by middleware and echoed in the `X-Correlation-Id` response header and in every log line for that request.

## Endpoint summary

| Method | Path | Auth required | Policy / role |
|---|---|---|---|
| POST | `/auth/login` | No | - |
| POST | `/auth/refresh` | No (refresh token in body) | - |
| GET | `/releases` | Yes | any authenticated user with `ReadPipeline` or `ReadAllStages` |
| GET | `/releases/{id}` | Yes | any authenticated user with `ReadPipeline` or `ReadAllStages` |
| POST | `/releases/{id}/assess` | Yes | policy `CanTriggerAssessment` (requires `TriggerAssessment` permission) |
| GET | `/releases/{id}/report` | Yes | any authenticated user with `ReadPipeline` or `ReadAllStages` |
| GET | `/releases/{id}/report/pdf` | Yes | any authenticated user with `ReadPipeline` or `ReadAllStages` |
| GET | `/health/live` | No | excluded from auth middleware |
| GET | `/health/ready` | No | excluded from auth middleware |

## POST /api/v1/auth/login

Authenticates a mock user and issues a JWT access/refresh token pair.

**Request**

```json
{
  "email": "coordinator@demo.io",
  "password": "Password123!"
}
```

**Response 200 OK**

```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "8f14e45f-ceea-4f6e-b1c3-3a1c1e6d9a2b",
  "expiresIn": 3600,
  "user": {
    "email": "coordinator@demo.io",
    "role": "ReleaseCoordinator",
    "permissions": ["ReadPipeline", "TriggerAssessment"]
  }
}
```

**401 Unauthorized** - invalid email/password. See error envelope below.

## POST /api/v1/releases/{id}/assess

**Headers**: `Authorization: Bearer <accessToken>` (must belong to `coordinator@demo.io`, `manager@demo.io`, or `admin@demo.io` - `qalead@demo.io` lacks `TriggerAssessment` and receives 403)

**Response 200 OK** (scripted demo dataset)

## GET /api/v1/releases/{id}/report

Returns the most recently generated `ReleaseReadinessResponse` for a release (same shape as the `assess` response above). Returns 404 if no assessment has run yet.

---

## GET /api/v1/releases/{id}/report/pdf

Returns the same report rendered as a PDF via `IReleaseReadinessReportFactory`. `Content-Type: application/pdf`, `Content-Disposition: attachment; filename="REL-2002-report.pdf"`.

---

## GET /api/v1/health/live and GET /api/v1/health/ready

Excluded from JWT middleware. No auth required.

**Response 200 OK**

```json
{ "status": "Healthy" }
```

`/health/ready` additionally checks dependency availability (mock data path readable, etc.) and returns `503 Service Unavailable` with the same shape if not ready.

---

## Error envelope

Every non-2xx response uses this shape:

```json
{
  "correlationId": "3f1e2b7a-9c4d-4e11-8a2f-6b7c8d9e0f1a",
  "statusCode": 403,
  "message": "You do not have permission to trigger an assessment.",
  "errors": ["Missing required permission: TriggerAssessment"],
  "timestamp": "2026-07-14T09:12:03Z"
}
```

### Status codes used

| Code | Meaning | Example cause |
|---|---|---|
| 400 | Bad Request | Malformed JSON body |
| 401 | Unauthorized | Missing, expired, or invalid JWT |
| 403 | Forbidden | Authenticated but missing required permission/role (e.g. `qalead@demo.io` calling `assess`) |
| 404 | Not Found | Unknown `releaseId` |
| 422 | Unprocessable Entity | FluentValidation failure (e.g. empty release id) |
| 429 | Too Many Requests | Rate limit exceeded (100 req/min/user) |
| 500 | Internal Server Error | Unhandled exception, caught by global exception middleware; stack trace never exposed |
| 503 | Service Unavailable | `/health/ready` when a dependency is unavailable |
